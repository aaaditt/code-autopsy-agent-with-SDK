# 🔬 Code Autopsy

**AI-powered GitHub repository analyzer built with Google ADK**

Paste any public GitHub repo URL → get a full Architecture map, Bug report, and Improvement Roadmap — generated by a multi-agent ADK pipeline powered by Gemini.

> Built for Google Gen AI Academy APAC 2025 — ADK Track

---

## Architecture

```
User → React UI → Flask API → ADK SequentialAgent Pipeline
                                 ├── ArchitectAgent     (maps structure)
                                 ├── BugFinderAgent     (finds issues)
                                 └── RoadmapAgent       (builds plan)
```

Each agent writes its output to `session.state` via `output_key`, so the next agent in the pipeline automatically has access to prior results.

## Local Setup

### 1. Backend

```bash
cd backend
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env
# Edit .env — add your GOOGLE_API_KEY
# Optionally add GITHUB_TOKEN for higher rate limits

python app.py
# API running at http://localhost:5000
```

### 2. Frontend

```bash
cd frontend
npm install

# Create .env.local
echo "VITE_API_URL=http://localhost:5000" > .env.local

npm run dev
# UI running at http://localhost:5173
```

## Deploy

### Backend → Render

1. Push to GitHub
2. New Web Service on [render.com](https://render.com)
3. Root directory: `backend`
4. Build command: `pip install -r requirements.txt`
5. Start command: `gunicorn app:app --bind 0.0.0.0:$PORT --workers 2 --timeout 120`
6. Add env vars: `GOOGLE_API_KEY`, `GITHUB_TOKEN` (optional)

### Frontend → Vercel

1. Import repo on [vercel.com](https://vercel.com)
2. Root directory: `frontend`
3. Framework preset: Vite
4. Add env var: `VITE_API_URL=https://your-render-url.onrender.com`
5. Deploy

## How it Works (ADK Pipeline)

```python
# 3 LlmAgents in a SequentialAgent
code_autopsy_pipeline = SequentialAgent(
    name="CodeAutopsyPipeline",
    sub_agents=[architect_agent, bug_finder_agent, roadmap_agent],
)
```

1. **Fetch** — `fetch_github_repo()` tool pulls the file tree + up to 25 key files from the GitHub API
2. **ArchitectAgent** — reads `repo_data` from session state, produces architecture report → writes to `architecture_report`
3. **BugFinderAgent** — reads `repo_data` + `architecture_report`, produces bugs report → writes to `bugs_report`
4. **RoadmapAgent** — reads all three, produces prioritized roadmap → writes to `roadmap_report`

Each agent's `output_key` auto-saves its response into `session.state`, which the next agent inherits automatically — this is core ADK pattern.

## API Reference

```
POST /analyze
Body: { "repo_url": "https://github.com/owner/repo" }
Returns: { repo, architecture_report, bugs_report, roadmap_report }

POST /fetch-repo
Body: { "repo_url": "..." }
Returns: { owner, repo, description, language, stars, file_count }

GET /health
Returns: { status: "ok" }
```
