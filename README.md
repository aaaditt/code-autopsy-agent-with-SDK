# 🔬 Code Autopsy

> **AI-powered GitHub repository analyzer built with Google ADK**

Paste any public GitHub repo URL → get a full **Architecture Map**, **Bug Report**, and **Improvement Roadmap** — generated by a multi-agent pipeline powered by Google ADK and Gemini 2.5 Flash, deployed on Google Cloud Run.

Built for **Google Gen AI Academy APAC Edition 2025 — Track 1: Build and Deploy AI Agents**.

## ✨ What it does

| Feature | Description |
|---|---|
| 🏗️ **Architecture Mapping** | Analyses file tree + source files → project structure, tech stack, data flow, key components |
| 🐛 **Bug & Security Analysis** | Finds logic errors, security vulnerabilities, antipatterns, performance issues — with file-level citations |
| 🗺️ **Improvement Roadmap** | Phased plan (Critical → Sprint → Quarterly) with health scores across code quality, security, architecture, docs |
| ⚡ **Zero Setup** | Paste a GitHub URL, get results in ~90 seconds. No installation, no account needed |
| 🔌 **REST API** | Clean JSON API consumable by any frontend or CI/CD pipeline |

## 🏗️ Architecture

```
User → React UI (Vercel)
           │
           ▼
    POST /analyze
           │
           ▼
  Flask API (Google Cloud Run)
           │
           ▼
  fetch_github_repo() tool
  ┌─────────────────────────┐
  │  GitHub API             │
  │  • File tree            │
  │  • Up to 25 key files   │
  └─────────────────────────┘
           │
           ▼
  ADK SequentialAgent Pipeline
  ┌─────────────────────────────────────────────┐
  │                                             │
  │  [1] ArchitectAgent                         │
  │      reads  → repo_data                     │
  │      writes → architecture_report           │
  │                    │                        │
  │  [2] BugFinderAgent                         │
  │      reads  → repo_data + architecture_report│
  │      writes → bugs_report                   │
  │                    │                        │
  │  [3] RoadmapAgent                           │
  │      reads  → repo_data + both reports      │
  │      writes → roadmap_report                │
  │                                             │
  └─────────────────────────────────────────────┘
           │
           ▼
  JSON Response → React UI renders 3 tabs
```

Each agent uses ADK's `output_key` to write its result into `session.state`, which the next agent reads automatically — the core ADK multi-agent pattern.

## 🚀 Local Setup

### Prerequisites

- Python 3.11+
- Node.js 18+
- A [Gemini API key](https://aistudio.google.com/apikey) (free)
- Optionally: a [GitHub Personal Access Token](https://github.com/settings/tokens) for higher rate limits

### 1. Clone the repo

```bash
git clone https://github.com/aaaditt/code-autopsy-agent-with-SDK.git
cd code-autopsy-agent-with-SDK
```

### 2. Backend setup

```bash
cd backend

# Create and activate virtual environment
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Edit .env and add your GOOGLE_API_KEY
```

Start the backend:
```bash
python app.py
# API running at http://localhost:5000
```

Test it:
```bash
curl -X POST http://localhost:5000/analyze \
  -H "Content-Type: application/json" \
  -d '{"repo_url": "https://github.com/pallets/flask"}'
```

### 3. Frontend setup

```bash
cd frontend
npm install
echo "VITE_API_URL=http://localhost:5000" > .env.local
npm run dev
# UI running at http://localhost:5173
```

## 🌐 Deployment

### Backend → Google Cloud Run

```bash
cd backend

gcloud auth login
gcloud config set project YOUR_PROJECT_ID
gcloud services enable run.googleapis.com cloudbuild.googleapis.com

gcloud run deploy code-autopsy-api \
  --source . \
  --region us-central1 \
  --allow-unauthenticated \
  --set-env-vars GOOGLE_API_KEY=your_key \
  --memory 1Gi \
  --timeout 300
```

### Frontend → Vercel

```bash
cd frontend
npm i -g vercel
vercel login
vercel --prod
# Add VITE_API_URL env var pointing to your Cloud Run URL
```

## 🧠 How the ADK Pipeline Works

```python
# Three specialised agents, each with a single responsibility
architect_agent = LlmAgent(
    name="ArchitectAgent",
    model="gemini-2.5-flash",
    output_key="architecture_report",  # writes to session state
)

bug_finder_agent = LlmAgent(
    name="BugFinderAgent",
    model="gemini-2.5-flash",
    output_key="bugs_report",
)

roadmap_agent = LlmAgent(
    name="RoadmapAgent",
    model="gemini-2.5-flash",
    output_key="roadmap_report",
)

# SequentialAgent runs them in order, passing state automatically
root_agent = SequentialAgent(
    name="CodeAutopsyPipeline",
    sub_agents=[architect_agent, bug_finder_agent, roadmap_agent],
)
```
